# 智能检测工厂产线系统 - 客户需求文档

## 1. 项目概述

本项目旨在为食品/药品包装产线构建一套智能检测与监控系统，实现对产线关键工艺参数的**实时采集、智能分析、异常告警和数据展示**，提升产线质量管控水平和生产效率。

## 2. 业务背景

- **行业领域**：食品/药品包装
- **检测目标**：包装完整性、密封性、标签准确性、异物检测、包装外观质量
- **核心痛点**：当前缺乏统一的数据监控平台，产线状态依赖人工巡检，异常发现滞后，缺乏历史数据追溯能力

## 3. 产线规模

| 维度 | 规格 |
|------|------|
| 产线数量 | 1-3 条产线 |
| 每线检测工位 | 3-5 个工位 |
| 每线数据采集点 | 5-15 个传感器数据点 |
| 数据频率 | 每秒数条（低频实时流式） |

## 4. 数据来源与协议

### 4.1 数据采集方式
- **主要来源**：PLC + 工业传感器
- **采集模式**：实时流式（低频，每秒 ≈ 1-5 条/产线）
- **通信协议**：标准工业协议（如 Modbus TCP / OPC UA / MQTT，由实现方根据模拟方案确定）

### 4.2 数据字段
每条产线需监控以下关键工艺参数：

| 参数类别 | 具体字段示例 | 单位 |
|----------|-------------|------|
| 温度/湿度 | 工位温度、工位湿度、热封温度 | ℃ / %RH |
| 压力/流量 | 气压、液压、充气压力、气体流量 | MPa / L/min |
| 速度/转速 | 传送带速度、主轴转速、生产节拍（件/分） | m/s / RPM / pcs/min |

### 4.3 模拟数据要求
- 系统需内置**模拟数据生成器**，可模拟上述传感器数据
- 模拟数据应在合理范围内波动，并偶尔生成异常值用于验证告警功能
- 模拟器可配置产线数量、数据点、采集频率等参数

## 5. 功能需求

### 5.1 数据采集模块
- [ ] 支持与 PLC/传感器设备的实时数据采集
- [ ] 内置模拟数据生成器（用于开发测试和无设备环境演示）
- [ ] 数据解析与标准化处理
- [ ] 采集状态监控（连接状态、采集速率）

### 5.2 数据分析模块

#### 5.2.1 实时异常检测
- [ ] 基于人工设定阈值的异常判断（上/下限阈值）
- [ ] 基于历史数据的动态基线分析（统计方法：均值 ± N×标准差）
- [ ] 多条件组合规则（如：温度超标 AND 速度异常）
- [ ] 异常等级分级（警告、严重、紧急）

#### 5.2.2 良品率 / OEE 统计
- [ ] 良品率实时统计（按产线/工位/时间段）
- [ ] OEE（设备综合效率）计算
- [ ] 不良原因分类统计

#### 5.2.3 趋势分析与报表
- [ ] 历史数据趋势图表（折线图、柱状图）
- [ ] 日报 / 周报 / 月报自动生成
- [ ] 自定义时间段数据查询与对比

### 5.3 数据展示模块

#### 5.3.1 大屏看板（核心展示终端）
- [ ] 产线整体态势总览（产品计数、良品率、OEE、设备状态）
- [ ] 实时数据流显示（滚动展示最新传感器数值）
- [ ] 实时趋势图（近N分钟温度/压力/速度曲线）
- [ ] 异常告警列表（未处理告警高亮显示）
- [ ] 产线工位拓扑示意图（每条产线的工位布局可视化）

#### 5.3.2 历史数据页面
- [ ] 历史数据查询（按产线/时间范围/参数类型筛选）
- [ ] 历史趋势对比图
- [ ] 数据导出（CSV/Excel）

#### 5.3.3 告警管理页面
- [ ] 告警规则配置（阈值设定、动态基线参数）
- [ ] 告警历史记录查询
- [ ] 告警确认/处理/关闭

## 6. 非功能需求

### 6.1 部署环境
- **部署方式**：纯内网部署（工厂本地服务器）
- **无外网依赖**：所有服务运行在局域网内，不依赖外部网络
- **并发访问**：支持 5-10 个大屏/浏览器同时访问

### 6.2 性能要求
- 数据采集延迟：< 1 秒
- 大屏数据刷新：≤ 2 秒
- 告警触发延迟：< 3 秒（从数据异常到告警展示）
- 历史数据查询响应：< 3 秒（7天内数据）

### 6.3 可靠性
- 系统 7×24 小时连续运行
- 数据存储保留至少 90 天
- 采集服务异常自动重连

### 6.4 可扩展性
- 架构预留 ML/AI 模型接入能力（未来可能引入智能缺陷识别）
- 支持产线数量和数据采集点的水平扩展

## 7. 约束条件

| 约束项 | 说明 |
|--------|------|
| 部署方式 | 纯内网，无外网访问 |
| 技术栈 | 由技术架构师推荐（优先考虑开源、内网友好的方案） |
| 数据库 | 由技术架构师推荐（需兼顾时序数据和业务数据） |
| 检测规则 | 人工阈值 + 动态基线（统计方法），暂不引入 ML |
| 人员配置 | 待定 |
| 项目周期 | 待定 |

## 8. 验收标准

1. 可完整模拟 1-3 条产线的传感器数据采集
2. 大屏看板实时展示所有数据点和告警信息
3. 异常数据在 3 秒内触发大屏弹窗+声音告警
4. 可自定义配置检测阈值
5. 可查询和导出历史数据
6. 系统稳定运行 72 小时无崩溃
7. 纯内网环境可独立部署运行
